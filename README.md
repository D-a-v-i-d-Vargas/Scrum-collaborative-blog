# Agile Blog Platform 🚀

Un sitio de blog corporativo, modular y responsivo desarrollado en un entorno colaborativo de alto rendimiento. Este proyecto demuestra la implementación práctica de arquitecturas front-end limpias combinadas con metodologías de desarrollo ágil.

---

## 📋 Descripción del Proyecto
El objetivo principal de este proyecto fue diseñar y desplegar una plataforma web multi-página robusta a partir de un backlog y un wireframe de diseño heredados. El sistema cuenta con un flujo de navegación unificado, una distribución de contenido optimizada semánticamente y componentes globales persistentes independientes del dispositivo.

### Características Principales:
* **Arquitectura Multi-Página:** Estructura modular dividida en secciones estratégicas: Inicio, Quiénes Somos y Contacto.
* **Navegación Persistente:** Menú de navegación global fijo y operativo a lo largo de toda la experiencia de usuario.
* **Enterprise Footer:** Pie de página homologado con control de metadatos del proyecto, autorías e información de contacto centralizada.
* **Diseño Visual Avanzado:** Estilización contemporánea mediante CSS para maximizar la legibilidad y el impacto visual del contenido multimedia.

---

## 🛠️ Tecnologías y Herramientas
* **Front-End Base:** HTML5 Semántico y CSS3 Avanzado.
* **Gestión de Versiones:** Git & GitHub Workflow (Ramas de tareas y Pull Requests).
* **Metodología:** SCRUM (Sprint Planning de 1 semana, estimación y priorización de Backlog).

---

## 📐 Flujo de Trabajo e Ingeniería del Proyecto
Para garantizar la entrega oportuna y la calidad del código bajo las restricciones del Sprint, el equipo operó bajo el marco de trabajo SCRUM. Cada requerimiento del cliente fue desglosado en un tablero Kanban dentro de GitHub Projects:

1.  **Planificación del Sprint:** Priorización de los componentes críticos de maquetación y navegación.
2.  **Aislamiento de Ramas:** Creación de ramas dedicadas por componente (`feature branches`) para evitar conflictos de integración.
3.  **Aseguramiento de Calidad:** Pruebas cruzadas de responsividad y consistencia de estilos antes del despliegue final.

---

## 🧠 Aprendizajes Clave (Retrospectiva del Sprint)
El desarrollo de esta plataforma nos permitió consolidar habilidades técnicas y operativas críticas para entornos profesionales:

* **Mitigación de Deuda Técnica:** Aprendimos a interpretar, adaptar y mejorar requerimientos ambiguos o diseños heredados de baja fidelidad ("plantillas deficientes") para entregar un producto de software premium.
* **Cohesión en Multi-páginas:** Dominamos la persistencia de componentes globales (Headers y Footers) a través de múltiples documentos HTML sin romper la cascada de estilos.
* **Colaboración Síncrona:** Perfeccionamos el flujo de trabajo en Git para resolver fusiones de código eficientemente dentro de un equipo multidisciplinario.
